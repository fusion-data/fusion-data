# AI Coding

## 使用要点

- 一次性上下文要足够完整
- 将“要求不确定性”前置到文档
- 把评审标准写进 Prompt

## 超级 Prompt 模型（摘）

你是资深全栈工程师，根据《XXX 开发指南》完成如下任务：
- 技术栈：Auxm + fusionsql/sqlx/sea-query + PostgreSQL（见文档）
- 输出：完整代码库、README.md、环境变量示例、seed 脚本、部署脚本
- 评审标准：可运行、无重大 lint/clippy 错误、端到端流程可用
- 约束：保持模块化、提供测试样例、关键设计附注释

## 步骤清单

1. 以创建一个网站为例，先定清楚目标与交付物
2. 和 ChatGPT 打磨一份超详细的网站开发掼
3. 指南应包含：
   - 业务模块：网站的所有功能
   - 页面描述：有多少页面，每页做什么
   - 技术规格：框架、ORM、数据库等
   - （PS：以上这些都可以让 ChatGPT 教你）
4. 反复打磨，直到你想不到还能改为止
5. 基于文档，让 ChatGPT 产出一份超级详细 Prompt
6. 把 Prompt 扔给 Codex 或 Claude Code，让它一把梭
7. 你可以先去做别的事
8. 回来后检查你的网站，按需迭代
9. 注意权限：给 Codex 或 Claude Code 足够的读写/仓库/部署权限
